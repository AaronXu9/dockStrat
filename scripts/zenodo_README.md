# CogLigandBench: Crystal-Structure Protein-Ligand Docking and Co-Folding Benchmark Data

This dataset contains docking poses, co-folding predictions, input preparation files, and RMSD/scoring tables generated by the CogLigandBench study. **Six docking methods** (AutoDock Vina, GNINA, ICM, ICM-RTCNN, SurfDock, UniDock2) and **four co-folding methods** (AlphaFold 3, Boltz-1, Chai-1, Protenix) were evaluated on the **runsNposes** benchmark (~1,280 protein-ligand systems from the PDB) using experimental co-crystal structures as both receptor input and ground truth.

**Associated code repository:** [https://github.com/AaronXu9/CogliandBench](https://github.com/AaronXu9/CogliandBench)

**Source benchmark dataset:** The runsNposes crystal structures are from the [Runs N' Poses](https://github.com/plinder-org/runs-n-poses) dataset and are **not** included in this deposit. Obtain them separately from the runs-n-poses repository.

## File Naming Conventions

### System identifiers

Most methods use the standard runsNposes naming: `{pdb_id}__{assembly}__{protein_chains}__{ligand_chains}`
- Example: `5s9l__1__1.A__1.H_1.I`

Exceptions:
- **ICM** replaces dots with underscores: `5s9l__1__1_A__1_H_1_I`
- **SurfDock** appends pocket chain info: `5s9l__1__1.A__1.H_1.I_8A_1.H`

### Docking pose file naming

| Method | Pattern | Ranking |
|--------|---------|---------|
| Vina | `{system}_pose{N}_score{S}.sdf` | Vina energy (most negative = best) |
| GNINA | `rank{N}_score{S}.sdf` | CNNscore (highest = best) |
| ICM | `rank{N}.sdf` | ICM score (lowest = best) |
| ICM-RTCNN | `RTCNN_rank{N}.sdf` | RTCNN confidence (highest = best) |
| SurfDock | `{pdb}_{chain}_{chain}.sdf_..._rank_{N}_rmsd_{V}_confidence_{C}.sdf` | Confidence (highest = best) |
| UniDock2 | `rank{N}.sdf` | Vina energy (most negative = best) |

## Results Table Columns

The CSV files in `results_tables/` (docking) and `cofolding_results_tables/` (co-folding) contain per-pose RMSD values and PoseBusters validation metrics. Key columns:

| Column | Description |
|--------|-------------|
| `method` | Method name |
| `rank` / `sample` | Pose rank / sample index |
| `rmsd` | RDKit CalcRMS symmetry-corrected RMSD vs crystal ligand |
| `kabsch_rmsd` | Kabsch-aligned RMSD |
| `centroid_distance` | Distance between predicted and crystal ligand centroids |
| `rmsd_<=_2a` | Boolean: RMSD at or below 2 Angstrom |
| `score` / `ranking_score` | Method-specific docking/confidence score |

Additional columns include PoseBusters molecular validity checks (bond lengths, angles, steric clashes, etc.).

## How to Use (Docking)

```python
from rdkit import Chem
from rdkit.Chem.rdMolAlign import CalcRMS

# Load crystal ligand (from runs-n-poses dataset, not included here)
ref = Chem.MolFromMolFile('data/runsNposes/5s9l__1__1.A__1.H_1.I/5s9l__1__1.A__1.H_1.I_ligand.sdf', removeHs=True)

# Load predicted pose
pose = Chem.MolFromMolFile('docking_poses/vina/5s9l__1__1.A__1.H_1.I/5s9l__1__1.A__1.H_1.I_pose1_score-10.97.sdf', removeHs=True)

# Compute symmetry-corrected RMSD
rmsd = CalcRMS(ref, pose)
print(f'RMSD: {rmsd:.2f} Angstrom')
```

## Co-Folding Methods

Unlike docking, which places a pre-built ligand into a fixed receptor pocket, **co-folding** models predict the full protein-ligand complex jointly from sequence and ligand SMILES. The receptor conformation is *not* held fixed — each prediction is an independent complex structure. Evaluation therefore requires (1) extracting the ligand from the predicted CIF complex and (2) aligning the predicted receptor to the crystal receptor before measuring ligand RMSD.

### Directory layout

```
cofolding_results_tables/
  predictions/                  RMSD and scoring per pose (one CSV per method)
  posebusters_results/          PoseBusters molecular validity per pose

cofolding_poses/
  {method}/{system}/
    ranking_scores.csv          seed, sample, ranking_score
    seed-{S}_sample-{N}.cif     full protein-ligand complex
```

Each system directory contains up to **5 CIF files** — the top-5 ranked poses selected from multiple seeds and samples per seed. Filenames encode **seed and sample number**, not rank. Rank is derived by joining with `ranking_scores.csv` (higher `ranking_score` = better pose).

### Loading the top-1 pose

```python
import pandas as pd

sys_dir = 'cofolding_poses/af3/5s9l__1__1.A__1.H_1.I'
df = pd.read_csv(f'{sys_dir}/ranking_scores.csv')
df = df.sort_values('ranking_score', ascending=False).reset_index(drop=True)
top1_cif = f"{sys_dir}/seed-{df.iloc[0]['seed']}_sample-{df.iloc[0]['sample']}.cif"
```

### Evaluating co-folding RMSD vs crystal

Because the predicted receptor conformation differs from the crystal structure, the canonical workflow is:

1. Parse the CIF complex (Biotite, BioPython, or gemmi).
2. Separate the ligand chain (non-polymer/heteroatom) from the protein.
3. Superpose the predicted receptor onto the crystal receptor (Kabsch on Calpha atoms or pocket residues).
4. Apply the same transform to the predicted ligand.
5. Compute symmetry-corrected ligand RMSD against the crystal ligand (RDKit `CalcRMS`).

Pre-computed RMSDs following this workflow are in `cofolding_results_tables/predictions/*.csv` (column: `rmsd`).

### Excluded variants

This deposit excludes `af3_no_template`, `boltz1x`, `boltz2`, and `rfaa` — these are model variants or alternate runs not used in the primary paper figures.

## Notes

- **ICM docking maps** (~8 GB raw, ~5.7 GB compressed) represent significant computational effort. Each system's map directory contains grid-based scoring fields used by the ICM docking algorithm. These are included to enable exact reproduction of ICM docking results without re-computing maps.

- **ICM `.icb` workspace files are excluded** from the docking poses. These are binary ICM session files (~5.7 MB each) that are not needed for result reproduction — the SDF output files contain the actual predicted poses.

- **PDBQT files** for Vina are the receptor and ligand files converted from PDB/SDF using OpenBabel with Gasteiger charges. GNINA reads PDB/SDF natively and does not require PDBQT input.

- **Co-folding CIF files** contain the full predicted protein-ligand complex. Expect ~550 KB per file. Compressed archives (~500-800 MB each) are provided for selective download per method.
