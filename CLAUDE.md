## Project Overview

**dockStrat** is a protein-ligand docking benchmarking framework designed specifically for **crystal/experimental PDB structures** (holo structures). It evaluates docking methods against ground-truth ligand poses from the PDB.

> **Paper:** Xu et al., *A Multi-Paradigm Benchmark of Molecular Docking: From Physics to Co-folding and Hybrid Models*. The paper benchmarks 9 of the supported methods (AF3, Boltz-1, Chai-1, Protenix, GNINA, ICM-RTCNN, ICM, Vina, Uni-Dock) on the runsNposes dataset stratified by SuCOS similarity to pre-2021 training data. Boltz-2 is implemented but excluded from the paper because its training set overlaps the test set.

Supported methods (all reachable via `dock_engine`): AutoDock Vina, Uni-Dock2, GNINA, ICM, ICM-RTCNN, DynamicBind, SurfDock, AlphaFold3, Chai-1, Boltz-1, Boltz-2, Protenix.

Primary benchmark dataset: **runsNposes** (~1,280 systems). Secondary: **plinder test set** (~1,038 systems).

> **Note:** Early development was seeded from [PoseBench](https://github.com/BioinfoMachineLearning/PoseBench). The project has since been extensively rewritten for crystal-structure workflows in the `dockstrat` package.

## Architecture

### Unified Docking API (`dock_engine`)

The primary entry point is `dock_engine` in `dockstrat/engine.py`:

```python
from dockstrat import dock_engine

# Dataset/batch mode — run over a full benchmark
dock_engine('vina',       dataset='runsNposes')
dock_engine('gnina',      dataset='runsNposes', repeat_index=1)
dock_engine('unidock2',   dataset='runsNposes', num_poses=20)

# Single-molecule mode — one protein-ligand pair
dock_engine('vina',       protein='receptor.pdb', ligand='ligand.sdf', output_dir='./out')
dock_engine('gnina',      protein='receptor.pdb', ligand='ligand.sdf', output_dir='./out')
dock_engine('chai',       protein='receptor.pdb', ligand='ligand.sdf', output_dir='./out',
            python_exec_path='/path/to/chai-lab/bin/python')
dock_engine('dynamicbind',protein='receptor.pdb', ligand='ligand.sdf', output_dir='./out')
dock_engine('unidock2',   protein='receptor.pdb', ligand='ligand.sdf', output_dir='./out')
dock_engine('surfdock',   protein='receptor.pdb', ligand='ligand.sdf', output_dir='./out')
dock_engine('boltz1',     protein='receptor.pdb', ligand='ligand.sdf', output_dir='./out')
dock_engine('boltz2',     protein='receptor.pdb', ligand='ligand.sdf', output_dir='./out')
dock_engine('protenix',   protein='receptor.pdb', ligand='ligand.sdf', output_dir='./out')
```

Kwargs are merged into the method's YAML config as overrides. In single-molecule mode, results land in `output_dir/{method}/{prefix}/`. In dataset mode, results go to the path defined in the method's YAML config.

### Configuration System (Hydra + OmegaConf)

Configs live in `dockstrat_config/model/<method>_inference.yaml`. All paths use `${oc.env:PROJECT_ROOT}` which is set at import time via `rootutils.find_root()` (anchored to `.project-root`). Custom OmegaConf resolvers are registered in `dockstrat/__init__.py`.

## Method Reference

All methods accept a `protein` PDB and `ligand` SDF as inputs. The table below summarizes the key differences.

All envs live under `envs/{method}/` (symlinks created by `scripts/install_{method}_env.sh`). ICM is physics-based and uses the commercial ICM binary on `$PATH` with no managed env.

| Method | Output format | Ranking metric | Env | Key dependency |
|--------|--------------|---------------|-----|----------------|
| `vina` | `rank{N}_score{S}.sdf` | Vina score (lowest best) | `envs/vina` | `obabel`, `vina` binary |
| `gnina` | `pose{N}_score{S}.sdf` | CNNscore (highest best) | `envs/gnina` (wraps `forks/GNINA/`) | `gnina` binary |
| `chai` | `rank{N}.sdf` | aggregate_score (highest best) | `envs/chai` | `chai_lab` package |
| `dynamicbind` | `rank{N}_ligand_lddt*.sdf` | DynamicBind rank | `envs/dynamicbind` (wraps `forks/DynamicBind/`) | `run_single_protein_inference.py` |
| `unidock2` | `rank{N}.sdf` | Vina energy (lowest best) | `envs/unidock2` | `unidock2` command |
| `surfdock` | `rank{N}.sdf` | Confidence (highest best) | `envs/surfdock` | MSMS, APBS, ESM, `accelerate` |
| `alphafold3` | `rank{N}.sdf` | AF3 ranking_score (highest best) | `envs/alphafold3` | AF3 source clone, decompressed `af3.bin` weights |
| `boltz1` | `rank{N}.sdf` | confidence_score (highest best) | `envs/boltz` | `boltz` CLI binary |
| `boltz2` | `rank{N}.sdf` | confidence_score (highest best) | `envs/boltz` | `boltz` CLI binary (+ affinity prediction) |
| `protenix` | `rank{N}.sdf` | ranking_score (highest best) | `envs/protenix` | `protenix` CLI binary |
| `icm` | `rank{N}.sdf` | ICM score | system | commercial ICM binary on `$PATH` |

## Per-Method Details

### Vina

**Input preprocessing:**
- Protein PDB → PDBQT via `obabel -i pdb -o pdbqt -xhk -r` (Gasteiger charges, rigid receptor)
- Ligand SDF → PDBQT via `obabel -i sdf -o pdbqt -xh` (Gasteiger charges)
- Binding box: computed from ligand centroid + protein neighbor analysis (MDAnalysis). Midpoint between ligand center and 50 closest protein atoms. Minimum 20 Å box with 10 Å margin.

**Outputs:** `{output_dir}/{prefix}_pose{rank}_score{score:.2f}.sdf` — one SDF per pose, ranked by Vina score (most negative = best). Also writes intermediate `protein.pdbqt`, `ligand.pdbqt`, `vina_out.pdbqt`.

**Post-processing:** Poses are ready-to-use SDF files. No further extraction needed.

**Requirements:**
- `obabel` (Open Babel) must be on `$PATH`
- `vina` binary must be on `$PATH`
- `MDAnalysis` (optional, improves binding site detection)
- Config: `dockstrat_config/model/vina_inference.yaml`

---

### GNINA

**Input preprocessing:**
- No user-side preprocessing required. GNINA reads PDB and SDF natively.
- Binding box: auto-detected from the reference ligand via `--autobox_ligand`.

**Outputs:** `{output_dir}/{prefix}_pose{rank}_score{score:.2f}.sdf` — one SDF per pose ranked by CNNscore (higher = better). Intermediate files: `docked.sdf.gz`, `docked.sdf`.

**Post-processing:** Poses are ready-to-use SDF files.

**Requirements:**
- Binary at `{PROJECT_ROOT}/forks/GNINA/gnina` (hardcoded relative to project root)
- GPU recommended (GNINA supports CPU but is much slower)
- Config: `dockstrat_config/model/gnina_inference.yaml`

---

### Chai-1

**Input preprocessing:**
- Protein sequence extracted from PDB via `dockstrat.utils.sequence.extract_protein_sequence` (CA atoms, biopandas, AA3→AA1 mapping with `X` for unknowns).
- Ligand SMILES extracted from SDF via RDKit (`Chem.SDMolSupplier` → `MolToSmiles`, canonical).
- A Chai-compatible FASTA is constructed:
  ```
  >protein|{prefix}-chain-{i}
  {sequence}
  >ligand|{prefix}-chain-{N+1}
  {smiles}
  ```
- This preprocessing happens internally in `run_single`. No user preparation needed.

**Outputs:** `{output_dir}/rank{N}.sdf` — top-N ligand poses extracted from the predicted PDB/CIF files and bond-order-recovered against the input SMILES via `rdkit.Chem.AllChem.AssignBondOrdersFromTemplate`. Ranked by Chai-1's `aggregate_score` from `scores.model_idx_*.npz` (highest score = `rank1.sdf`). Intermediate Chai outputs (`pred.model_idx_*.{cif,pdb}`, `scores.model_idx_*.npz`, `{prefix}.fasta`) remain in `{output_dir}/` for post-mortem.

**Post-processing:** None — extraction is inline. SDFs are ready for downstream RMSD analysis.

**Requirements:**
- Conda env with `chai_lab` installed. Python executable path configured via `python_exec_path` in YAML or passed as kwarg:
  ```python
  dock_engine('chai', protein=..., ligand=..., output_dir=...,
              python_exec_path='/path/to/forks/chai-lab/chai-lab/bin/python')
  ```
- `chai_lab` package must be installed in that env (`from chai_lab.chai1 import run_inference`)
- GPU with ≥24 GB VRAM recommended. Proteins >~500 residues may OOM on a 24 GB card.
- Config: `dockstrat_config/model/chai_inference.yaml`
  - `cuda_device_index`: GPU to bind via `CUDA_VISIBLE_DEVICES`
  - `num_trunk_recycles` (3), `num_diffn_timesteps` (200), `seed` (42), `use_esm_embeddings` (true): Chai-1 inference knobs
  - `num_poses_to_keep`: how many top-ranked SDFs to write per system (default 5)
  - `timeout_seconds`: per-system hard cap (default 3600)

### DynamicBind

**Input preprocessing:**
- SMILES extracted from the input SDF via RDKit
- A temporary CSV is created: `ligand\n{smiles}` — this is DynamicBind's native input format
- No protein preprocessing; DynamicBind reads the PDB directly

**Outputs:** `{output_dir}/rank{N}_ligand_lddt{lDDT}_affinity{aff}.sdf` — SDF files for up to 40 samples (configurable via `samples_per_complex`). Ranked by DynamicBind's internal scoring. Also caches intermediate data in `{PROJECT_ROOT}/data/dynamicbind_cache/`.

**Post-processing:** Poses are ready-to-use SDF files. The lDDT and affinity scores are encoded in the filename.

**Requirements:**
- Python interpreter at `{PROJECT_ROOT}/forks/DynamicBind/DynamicBind/bin/python3`
- DynamicBind script: `{PROJECT_ROOT}/forks/DynamicBind/run_single_protein_inference.py`
- `--no_relax` and `--paper` flags are hardcoded (relaxation not supported; only paper weights available)
- GPU required
- Config: `dockstrat_config/model/dynamicbind_inference.yaml`

### UniDock2

**Input preprocessing:**
- Ligand centroid computed from SDF atom coordinates via RDKit (mean of all atom positions)
- A docking config YAML is generated with `center`, `size`, `search_mode`, `num_poses`, `energy_range`
- Default box: 15 × 15 × 15 Å centered on ligand centroid. Override via `box_size` kwarg.

**Outputs:** `{output_dir}/rank{N}.sdf` — one SDF per pose, ranked by Vina binding free energy (lowest = best). Also writes `{prefix}_poses.sdf` (raw multi-pose output).

**Post-processing:** Poses are ready-to-use SDF files.

**Requirements:**
- `unidock2` conda env must be available (`conda run -n unidock2 unidock2 docking -cf {config}`)
- GPU required
- Config: `dockstrat_config/model/unidock2_inference.yaml`
  - `box_size`: [15.0, 15.0, 15.0] (Å, adjustable)
  - `search_mode`: `detail` (or `fast`, `balance`)
  - `energy_range`: 15.0 kcal/mol window
  - `timeout`: 1800 s per system

### SurfDock

**Input preprocessing (4-step pipeline, all in a temp dir):**

1. **Surface computation** (`_surfdock_surface_helper.py`)
   - Crops pocket residues within 8 Å of ligand
   - Computes MSMS triangulated surface (requires `msms` binary)
   - Computes APBS electrostatics (requires `apbs`, `pdb2pqr`, `multivalue` binaries)
   - Produces: `{prefix}_protein_processed_8A.pdb` + `.ply`

2. **Input CSV construction** (`inference_utils/construct_csv_input.py`)
   - Collects protein path, pocket PDB, reference ligand, ligand SDF, surface PLY
   - Produces: `input.csv`

3. **ESM pocket embeddings** (3 sub-steps)
   - Extract FASTA from CSV → run ESM2-3B (`esm2_t33_650M_UR50D`) → crop to pocket → merge into `.pt`
   - Produces: `esm2_3billion_pdbbind_embeddings.pt`

4. **Diffusion inference** (`accelerate launch inference_accelerate.py`)
   - Runs the SurfDock diffusion model
   - Produces: `SurfDock_docking_result/{pocket}_{ligand}/{name}_rank_{R}_confidence_{C}.sdf`

**Outputs:** `{output_dir}/surfdock/{prefix}/rank{N}.sdf` — one SDF per pose, ranked by confidence (rank1 = highest). `num_poses` controls how many are kept (default 40).

**Post-processing:** Poses are ready-to-use SDF files.

**Requirements:**
- **Conda env**: `SurfDock` (`/home/aoxu/miniconda3/envs/SurfDock`)
- **`surfdock_dir`** must point to the **local SurfDock installation** (`/home/aoxu/projects/SurfDock`), NOT the git submodule, because:
  - MSMS/APBS/pdb2pqr binaries (231 MB) are extracted only in the local installation (`comp_surface/tools/`)
  - ESM library is installed as an editable install from the local directory
  - The git submodule at `forks/SurfDock/` only contains the code and model weights
- **`precomputed_arrays`** env var must be set to `/home/aoxu/projects/precomputed/precomputed_arrays` (required at import time by SurfDock's `utils/torus.py` and `utils/so3.py`)
- **Model weights** are served from `forks/SurfDock/model_weights/` (in the submodule, already committed)
- GPU required (`accelerate launch` with `--num_processes=1`)
- Config: `dockstrat_config/model/surfdock_inference.yaml`
  - `surfdock_dir`: `/home/aoxu/projects/SurfDock` (local install — do not change to submodule path)
  - `precomputed_arrays`: `/home/aoxu/projects/precomputed/precomputed_arrays`
  - `diffusion_model_dir`: `${PROJECT_ROOT}/forks/SurfDock/model_weights/docking`
  - `confidence_model_dir`: `${PROJECT_ROOT}/forks/SurfDock/model_weights/posepredict`

**Dataset mode:** For the runsNposes benchmark, Steps 1–3 are pre-computed. `run_dataset` skips straight to inference using:
- `inputs_csv`: pre-built input CSV
- `esm_embeddings_path`: pre-computed `.pt` embeddings
- `surface_dir`: pre-computed 8 Å surface files

### AlphaFold3

**Input preprocessing:**
- Protein chain sequences extracted from the PDB via `dockstrat.utils.sequence.extract_protein_sequence` (CA atoms, biopandas, AA3→AA1 mapping with `X` for unknowns).
- Ligand SMILES extracted from the SDF via RDKit (`Chem.SDMolSupplier` → `MolToSmiles`, canonical).
- An AF3 input JSON is built per system: one `protein` entry per chain (chain ids `A`, `B`, ..., each with a single-sequence A3M as `unpairedMsa`, empty `pairedMsa`, empty `templates`) and one `ligand` entry on chain `L` carrying the canonical SMILES. `dialect: "alphafold3"`, `version: 2`, `modelSeeds: [1234]`.
- This JSON is written to `{output_dir}/{prefix}_af3_input.json` before each run for inspection.

**Outputs:** `{output_dir}/rank{N}.sdf` — top-N ligand poses extracted from the predicted mmCIFs and bond-order-recovered against the input SMILES via `rdkit.Chem.AllChem.AssignBondOrdersFromTemplate`. Ranked by AF3's own `ranking_scores.csv` (highest score = `rank1.sdf`). Intermediate AF3 outputs (full mmCIFs, confidence JSONs) remain in `{output_dir}/{prefix_lowercase}/` for post-mortem.

**Post-processing:** None — extraction is inline. SDFs are ready for downstream RMSD analysis.

**Requirements:**
- Conda env at `{PROJECT_ROOT}/envs/alphafold3` (symlink to `/mnt/katritch_lab2/aoxu/envs/alphafold3`). Created by `bash scripts/install_alphafold3_env.sh`.
- AF3 source clone at `{PROJECT_ROOT}/forks/alphafold3/alphafold3` (created by the same install script).
- Decompressed weights at `{PROJECT_ROOT}/forks/alphafold3/models/af3.bin` (decompressed from `af3.bin.zst` by the install script).
- GPU required. `num_recycles=10`, `num_samples=5` on the default config; reduce for faster smoke tests.
- Runs in **single-sequence / no-MSA mode** — no MSA databases, no `db_dir`. Inference uses `--norun_data_pipeline`.
- Config: `dockstrat_config/model/alphafold3_inference.yaml`
  - `cuda_device_index`: GPU to bind via `CUDA_VISIBLE_DEVICES`
  - `num_samples` (mapped to AF3's `--num_diffusion_samples`), `num_seeds`, `num_recycles`: AF3 inference knobs. Exact CLI flag spellings may drift across AF3 releases; reconcile against `run_alphafold.py --help` after install.
  - `num_poses_to_keep`: how many top-ranked SDFs to write per system
  - `timeout_seconds`: per-system hard cap (default 3600)

### Boltz-1 / Boltz-2

**Input preprocessing:**
- Protein chain sequences extracted from PDB via `extract_protein_sequence()` (same utility as AF3/Chai).
- Ligand SMILES extracted from SDF via RDKit.
- A Boltz-compatible YAML is constructed with `version: 1`, protein entries (one per chain, `msa: "empty"` for single-sequence mode), and a ligand entry with canonical SMILES.
- Chain IDs are assigned sequentially: A, B, ... for protein chains, next letter for the ligand.
- For Boltz-2, a `properties: [{affinity: {binder: <ligand_chain>}}]` section is added to request affinity prediction.

**Outputs:** `{output_dir}/rank{N}.sdf` — top-N ligand poses extracted from predicted mmCIF files and bond-order-recovered against the input SMILES. Ranked by Boltz's `confidence_score` from `confidence_{stem}_model_{N}.json` (highest = `rank1.sdf`). For Boltz-2, affinity predictions (`affinity_pred_value`, `affinity_probability_binary`) are attached as SDF properties. Intermediate Boltz outputs (mmCIFs, confidence JSONs, processed data) remain in `{output_dir}/boltz_results_{prefix}/` for post-mortem.

**Post-processing:** None — extraction is inline. SDFs are ready for downstream RMSD analysis.

**Requirements:**
- Conda env with `boltz` CLI installed. Binary path configured via `boltz_binary` in YAML or kwarg.
  Default: `${PROJECT_ROOT}/envs/boltz/bin/boltz`. Install via `bash scripts/install_boltz_env.sh`.
- GPU required.
- Config: `dockstrat_config/model/boltz1_inference.yaml` and `boltz2_inference.yaml`
  - `model`: `"boltz1"` or `"boltz2"` (selects model variant)
  - `cuda_device_index`: GPU to bind via `CUDA_VISIBLE_DEVICES`
  - `diffusion_samples` (5), `recycling_steps` (3), `sampling_steps` (200), `seed` (42): inference knobs
  - `num_poses_to_keep`: how many top-ranked SDFs to write per system (default 5)
  - `timeout_seconds`: per-system hard cap (default 3600)
  - Boltz-2 only: `sampling_steps_affinity` (200), `diffusion_samples_affinity` (5), `affinity_mw_correction` (false)

### Protenix

**Input preprocessing:**
- Protein chain sequences extracted from PDB via `extract_protein_sequence()`.
- Ligand SMILES extracted from SDF via RDKit.
- A Protenix-compatible JSON array is constructed: `[{"name": system_id, "sequences": [{"proteinChain": {"sequence": ..., "count": 1}}, {"ligand": {"ligand": SMILES, "count": 1}}]}]`.
- Protenix uses `proteinChain` (not `protein`) and `ligand.ligand` for the SMILES field.
- Omitting MSA fields triggers single-sequence mode.

**Outputs:** `{output_dir}/rank{N}.sdf` — top-N ligand poses extracted from predicted mmCIF files and bond-order-recovered against the input SMILES. Ranked by `ranking_score` from `ranking_scores.csv` (highest = `rank1.sdf`). Intermediate outputs remain in `{output_dir}/{prefix}/` for post-mortem.

**CIF chain naming:** Protenix uses `{letter}0` format chain IDs: `A0`, `B0`, `C0`. Ligand atoms have `group_PDB = HETATM`. The ligand chain is discovered automatically via HETATM scanning with bond-order recovery fallback.

**Post-processing:** None — extraction is inline. SDFs are ready for downstream RMSD analysis.

**Requirements:**
- Conda env with `protenix` installed (`pip install protenix`). Requires Python 3.11+.
  Default binary: `{PROJECT_ROOT}/envs/protenix/bin/protenix`
- GPU required. Install script: `bash scripts/install_protenix_env.sh`
- Models auto-download to `~/.protenix` on first run.
- Config: `dockstrat_config/model/protenix_inference.yaml`
  - `model_name`: `"protenix-v2"` (default) or `"protenix_base_default_v1.0.0"`
  - `cuda_device_index`: GPU to bind via `CUDA_VISIBLE_DEVICES`
  - `seeds`: comma-separated seed list (default `"42"`)
  - `num_poses_to_keep`: how many top-ranked SDFs to write per system (default 5)
  - `timeout_seconds`: per-system hard cap (default 3600)

## Analysis

After docking, compute RMSD vs crystal structure using RDKit's symmetry-corrected RMSD:

```python
from rdkit import Chem
from rdkit.Chem.rdMolAlign import CalcRMS

ref = Chem.MolFromMolFile('crystal_ligand.sdf', removeHs=True)
pose = Chem.MolFromMolFile('rank1.sdf', removeHs=True)
rmsd = CalcRMS(ref, pose)  # symmetry-corrected
```

For Chai-1, the ligand must first be extracted from the complex PDB. Use `dockstrat/data/chai_output_extraction.py`.

Batch analysis scripts: `dockstrat/analysis/inference_analysis.py`.
